Intro XML PATH

A SELECT query returns results as a row set. You can optionally retrieve formal results of a SQL query as XML by specifying the FOR XML clause in the query. The FOR XML clause can be used in top-level queries and in subqueries. The top-level FOR XML clause can be used only in the SELECT statement. In subqueries, FOR XML can be used in the INSERT, UPDATE, and DELETE statements. FOR XML can also be used in assignment statements.

How to use XML PATH

With the XML PATH，You can append multiple Synonym names to the list from any one rule by separating each one with a separator. But in the actual development, there are also some disadvantages. Too many Synonym Names can cause display problems. It also becomes more cumbersome for the front end to implement data changes to the column. In addition, the statement of SQL has been changed, which is also bad for maintenance. Eventually, I put the data manipulation in the Front-End.

PDLP - Product Developer Leadership Program Overview

The purpose of the Product Developer Leadership Program (PDLP) is to mentor and train team members to become proficient in all core technologies, processes and standards utilized at Gensuite. Mentors will advise developers through hands-on training exercises; while simultaneously introducing developers to the products they will support. Graduating team members will then be able to execute on projects and product support with minimal technical guidance on basic to intermediate requests.

Program Areas

Each program area below is a starting point and guide to Gensuite Processes, Architecture and programming concepts. Review with your mentor, explore additional content online (e.g. Adobe CF Docs for example) and execute on the mentioned exercises together with assigned coding exercises from your mentor.

  • Gensuite Processes
  • User Interface Design
  • Client-side Scripting
  • Server-side Scripting
  • Database Programming
  • Platform Architecture

Developers

  • Explore each program area and code examples of each standard introduced to demonstrate understanding of each development standard.
  • Review code with Mentors to receive feedback on best practices or ask questions.
  • Be able to explain processes and architecture to your Mentor.

Mentors

  • Coach developers on coding standards, platform architecture and team processes via a daily touch point.
  • Assign exercises mentioned in PDLP documents and additional coding assignments as needed.
  • Work with Unit and Group Leaders on training strategies and transition into product support.

Intro

Flatpickr is a lightweight and powerful datetime pickr. It is integrated into company’s custom Tag.

Background

Why did I use it? Since I got a customer’s request in one App from Amazon. Amazon has a lot of Data, what they need is that adding a date limitation on searching filter.

What I did was limit the two inputs to 30 days. Not just a single control, but both will limit each other. For instance, When you choose a exact date in the former one, Then the latter one will have a 30-days limitation. And when you choose a exact date in latter one, the latter one has no limitation and the former one is limited.

I Looked through the FlatPickr Documentation. No similar examples can be found. With the help of my mentor, I figured it out. It is not enough to determine whether the input is empty. There’s always a case it cannot be implemented.We need add extra logic to make it happen.
